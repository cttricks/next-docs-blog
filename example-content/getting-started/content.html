Welcome to Your New Blog

This guide will walk you through everything you need to know about creating and managing content in your file-based blog system.

Content Structure

Each blog article consists of two files in a dedicated folder:

  • content.html - Your article content in pure HTML
  • meta.json - Metadata including title, description, and SEO information

Creating a New Article

To create a new article, follow these steps:

  1. Create a new folder in your content directory with a URL-friendly name (e.g., my-awesome-article)
  2. Add a meta.json file with required metadata
  3. Add a content.html file with your article content
  4. Your article is now accessible at /blogs/my-awesome-article

The meta.json File

Here's an example of a complete meta.json file:

{
  "title": "Your Article Title",
  "description": "A brief description for SEO",
  "ogTitle": "Social Media Title",
  "ogDescription": "Social media description",
  "ogImage": "https://example.com/image.jpg",
  "author": "Your Name",
  "publishedAt": "2024-02-14T00:00:00.000Z"
}

Required Fields

  • title - The main title of your article
  • description - A brief description for SEO and previews

Optional Fields

  • ogTitle - Custom title for social media sharing
  • ogDescription - Custom description for social media
  • ogImage - Image URL for social media cards
  • author - Author name
  • publishedAt - Publication date in ISO format

Writing Content

Your content.html file should contain pure, static HTML. Here are some tips:

Write semantic HTML using proper heading hierarchy, paragraphs, lists, and other standard elements. The system will automatically style your content beautifully.

Supported HTML Elements

You can use all standard HTML elements:

  • Headings: <h1> through <h6>
  • Paragraphs: <p>
  • Lists: <ul>, <ol>, <li>
  • Links: <a href="...">
  • Images: <img src="..." alt="...">
  • Quotes: <blockquote>
  • Code: <code>, <pre>
  • Emphasis: <strong>, <em>

Performance and Caching

The blog system is optimized for performance:

  • Server-Side Rendering - Articles are rendered on the server for fast initial loads
  • Static Caching - Pages are cached for 1 hour by default
  • On-Demand Revalidation - Update cached content without rebuilding the entire site

Revalidating Content

When you update an article, you can trigger a revalidation to update the cached version:

curl -X POST http://yoursite.com/api/revalidate \
  -H "Content-Type: application/json" \
  -d '{"slug": "your-article-slug", "secret": "your-secret-key"}'
